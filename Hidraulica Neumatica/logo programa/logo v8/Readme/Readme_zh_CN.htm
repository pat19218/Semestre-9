概述

安装和LOGO! Soft Comfort的启动程序在较旧的计算机上会需要较长的时间，并且在此时间内计算机屏幕上不会显示任何东西。严禁反复多次启动应用程序。

安装

由于Java并不总能正确地识别操作平台专有的字符，因此在目录名中请避免使用诸如感叹号（！）、右斜线（/）、冒号（：）、分号（；）、左斜线（\）、句号（。）、问号（？）、单引号（’）、双引号（”）等特殊字符。

您可以通过将下列步骤1提及的目录中的内容拷贝至硬盘上然后安装LOGO! Soft Comfort。MS-Windows操作系统中可执行程序的名称为LOGOComfort.exe，Mac-OSX操作系统中此程序的名称为LOGOComfort.app，Linux操作系统中为LOGOComfort。

按照如下步骤将LOGO!Soft Comfort复制到服务器上：

  1. 将LOGO!Soft Comfort DVD光盘上下列目录中的内容拷贝至服务器上：

l         <dvdrom>: \Windows\Application_32 或 \Windows\Application_64

l         <dvdrom>: \Mac\Application

l         <dvdrom>: \Linux\Application_32 或 \Linux\Application_64  

  1. 创建一个从客户机端到本服务器目录的连接。

如果您已经通过安装程序将LOGO!Soft Comfort安装在服务器上，那么您必须将.lax文件的所有绝对路径条目（例如：lib\classes.jar）更改为与安装条目相关的路径。

您也可以从上面提到的LOGO!Soft Comfort DVD光盘里的应用程序文件夹中复制此文件。

卸载

您可以使用卸载程序卸载各种操作平台上的LOGO!Soft Comfort。例如，在MS-Windows操作平台上，此卸载程序位于：

<安装目录>\UninstallerData\Uninstall.exe.

其他语言

在默认情况下，LOGO!Soft Comfort支持六种语言。 默认语言为安装过程中选中的语言。可以通过以下方式来转换语言：

  1.     在菜单栏中，点击“工具(T)”à“选项(O)”。
  2.     在“选项框中选择需要的语言，并点击“确定”。
  3.     重启LOGO!Soft Comfort软件。

已知问题

1.             如果您是在MS-Windows即插即用操作系统上操作带有LOGO!Soft Comfort的LOGO!硬件，那么会出现通讯错误。为解决此问题，可以按照下列步骤尽量缩短通讯端口的FIFO缓冲时间长度：

1)      点击“属性”，然后选择端口设置表；

2)      点击“高级”，然后将接收缓冲器和发送缓冲器设置为“低”。

2.      若无法打印到页面边界，可以通过“文件 > 页面设置”来调整页边空白。

3.      使用Kyocera FS1600打印机打印会损坏图片中的文本。为解决此问题，需安装“Kyocera FS1600+ （KPDL2）”打印机驱动。 在CD光盘上有一个英文版本的打印机驱动，路径为<dvdrom>:/windows/printer/k9xpsen.exe。

有关安装此打印机驱动的详情，请参考操作系统手册。

4.      如需解决打印机问题，请更新打印机驱动或者从操作系统CD光盘中安装标准的驱动。

5.      如果还是存在打印问题，则您可以将文件保存为PDF格式，然后从Adobe Acrobat Reader中打印。您还可以将图片保存为BMP或者JPG格式，或者在其他应用程序中进行编辑。

6.      对于光标在模拟模式下忽隐忽现的问题，可以禁止用户专属鼠标设置或者鼠标跟踪。

7.     对于windows操作系统，如果安装路径的最后一级目录名称中包含“空格”，系统会报错启动LOGO!Soft Comfort 失败。为解决此问题，请确认在安装路径的最后一级目录名称中不包含“空格”。

Macintosh

前提条件

您已经完成对从（或至）带有Keyspan USB PDA适配器（Keyspan USA-19）和LOGO! USB PC线缆的LOGO!硬件进行的上载操作（或下载操作）测试。

如果LOGO!Soft Comfort 无法通过同Keyspan USA-19或者LOGO! USB PC线缆对应的串行端口与LOGO!基本模块进行通讯，请检查文件系统中是否存在一个/var/lock文件夹。如果该文件夹不存在，请创建此文件夹并确保您享有访问此文件夹的完全权利。如果您以前使用过一个较旧版本的RXRXComm文件包，那么您还需要将旧有/var/spool/uucp文件夹中的剩余闭锁文件全部删除。在运行安装应用程序之前请检查这些文件。

安装

如果您无法执行Setup_Mac.command安装程序，那么有可能您无权进行此操作。例如，如果Setup_Mac.command程序是您从一台Windows操作系统的计算机上拷贝来的，那么你就会无法运行此程序。

按照下列操作来对Setup_Mac文件的用户属性进行变更：

在您试图启动Setup_Mac.command程序时，您会接收到一个“打开命令文件…”错误消息。确认此错误消息。在下一个终端窗口中，首先根据提示输入下列命令以更改用户属性：

chmod a+x Setup_Mac.command

然后，输入./Setup_Mac.command字符串来启动安装程序。也可以关闭终端窗口，然后双击可执行程序来启动安装程序。（注意：文件名区分大小写）

打印

MAC OS-X操作系统不支持横向打印。您如果需要这种格式的硬拷贝，可以先使用“另存为”菜单命令来将此图片保存为PDF格式，然后进行打印。
